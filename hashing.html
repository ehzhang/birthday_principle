Hash Tables and Birthdays

What is a Hash Table? What is a Hash Function?
What does this have anything to do with birthdays?

A hash table is a data structure important in computer science that stores data, known as a key as a hash value. We turn a key into a hash value by applying some kind of hash function.


Example of hashing keys. Source: http://en.wikipedia.org/wiki/Hash_function

We can think of hashing as placing data (such as a word) into a certain number of "buckets." For example, John Smith hashes to bucket 2, and Lisa Smith hashes to bucket 1.

Suppose we wanted to hash a set of integers into n buckets. We would want to use a hash function to map each integer to one of the n buckets. For example, one simple hash function is taking an an integer input mod n, where the modulus operator is the remainder of the integer when divided by n. This is called the 'division method' for hashing.

A collision occurs when two different inputs result in the same hash value. For example, 6 mod 5 and 11 mod 5 both result in '1'.

Using the modulus operator allows us to restrict the overall size of the space used for the hash table. Because the modulus operator is a remainder, for 'n' buckets there cannot be a hash value greater than 'n'.

A good number would be a large prime number so it is less likely for multiple numbers to be divisible by it. A terrible number would be a number like '2', which would likely result in many collisions.

The main reason why hash tables are so prevalent in computer science is because they provide fast lookup time. The time required for us to check if an item exists in the hash table does not depend on the amount of items in the table. Suppose we wanted to check whether "John Smith" was in our table. We could quickly compute the hash value for John Smith and check to see if the corresponding bucket is empty or not.

How can we reduce collisions with hash functions?

One method to hash would be to strive towards a hash function that would have a uniform distribution of values into buckets, so that each slot in the hash table has an equal opportunity of being filled. This would follow from the Simple Uniform Hashing Assumption, or 'SUHA' for short, which assumes that there is some hash function that is uniformly distributed, to allow for the mathematical analysis of hash tables.

With the Simple Uniform Hashing Assumption (SUHA), we assume that the probability for distribution of values to 'buckets' is uniform, much like we assumed that the probability for every birthday is a uniform distribution.

A hash table acts like a variant of the birthday principle problem, where 'birthdays' represent a hash table of size 365, one for each birthday. As the probability of every birthday is equal, like in the hashing assumption, we can relate the probability of a single hash table collision to the probability of a shared birthdays in a room.

Thus, we can use the same equation from the birthday principle, modified so we can get the probability of a hash table collision in a table of size 'M' as:
P('SingleCollision') = 1 - e^{-\frac{n(n-1)}{2(M)}}

Where 'n' is the number of values that have been added to the hash table.
